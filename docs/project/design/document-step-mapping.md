# Document Step Mapping

Each project step annotated with the document it produces (if any).

- What is the problem?
- Do existing solutions exist?
- Brainstorm ways to solve it -> (topic)-brainstorm.md, changelog.md (appended)
- Research brainstormed solutions -> (topic)-research.md, changelog.md (appended)
- Define and document chosen solutions and why? -> project-requirements.md, changelog.md (appended)
- Define and document chosen technology and why? -> project-techstack.md, changelog.md (appended)
- Focused in-depth research on the chosen technology -> (topic)-research.md (appended), changelog.md (appended)
- Design the solution -> project-design.md, project-file-org.md, changelog.md (appended)
- Break the chosen solution down into its main features -> feature-mapping.md (partial), changelog.md (appended)
- Map out if any features are dependent on each other -> feature-mapping.md (complete), changelog.md (appended)
- Create high-level project roadmap -> project-roadmap.md, changelog.md (appended)
- Break the roadmap features down into the smallest logical tasks -> feature-task-mapping.md (partial), changelog.md (appended)
- Map out if any tasks are dependent on each other -> feature-task-mapping.md (complete), changelog.md (appended)
- Create feature-specific, dependency-organized task lists -> feature-tasks.md, changelog.md (appended)
- Assign tasks to the appropriate specialist -> changelog.md (appended)
- Execute each task -> feature-tasks.md (checked off), changelog.md (appended)
- Commit changes -> changelog.md (appended)
- Review each task's output -> changelog.md (appended)
- Categorize any issues found by severity -> feature-todos.md, changelog.md (appended)
- Decide what to fix and what to defer -> changelog.md (appended)
- Apply fixes -> feature-tasks.md (checked off), changelog.md (appended)
- Commit changes -> changelog.md (appended)
- Re-review fixed tasks -> feature-todos.md (updated), changelog.md (appended)
- Test the implementation -> changelog.md (appended)
- Repeat the review cycle if issues are found -> feature-todos.md (updated), changelog.md (appended)
- Final commit -> changelog.md (appended)
- Document what was done and why -> changelog.md (appended)
